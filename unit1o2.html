Review of IT tools.

2.1. Visual Studio Code or VSC for short is designed by Miscrosft as a cross-platform code editor for writing on the web and cloud application. It is one of the most popular source code editors used by programmers. It is fast, it is lightweight and it is powerful. I am happy using VSC because I find it easy to use and it provides live coding regardless of the programming language.

Trello is a wonderful tool for managing tasks and projects. The visual Kanban boards are flexible, shareable, and let you pack a ton of detail into each card. Trello is also good for organizing just about anything, even my entire life.

2.2. The IT tools that were selected were appropriate for my project. They minimized confusion around project assignments with simple tools to organize tasks and track progress. They offered accurate and live coding.

2.3. The strengths of my final project are that I got to learn a lot in such a short period of time. I am proud of what I have achieved and I like the look of my website. The weaknesses are time constraits. I feel that the final project was too wordy for the time we had to do it in and I felt that I was doing more typing that designing. If I had more time, I would have spent it on designing the website and probably do a much better job especially incorporating some of the effects we were taught. I would have liked to insert more pictures too.

2.4 I am still not confident with the code so everything I have been doing has been trial and error. The mistake I made was to do the write up before designing the skeleton of the website because that took up a lot of my time. The improvements I would make on my website is to make it more interactive and responsive. Right now it is too wordy!

3.1. The benefits of IT tools and systems in terms of productivity and efficiency that

  1. They minimise confusionaround project assignments.
  2. Help organize tasks and track progress.
  3. Customizable lists that allow users to follow metrics they are most interested in and use automated notifications to keep informed of all changes.

3.2. There are only so many hours in the day, so making the most of your time is critical. The two ways to improve productivity and efficieny is to either put in more hours or work smarter. I don't know about you, but I prefer the latter. The way to do this is by;

  1. Tracking and limiting how much time you're spending on tasks.
  2. Taking regular breaks.
  3. Set yourself deadlines.
  4. Do not multitask.
  5. Minimise interruptions.
  6. Be proactive, not reactive

3.3. What I would do differently next time to improve productivity using IT in digital projects is:

  1. Organize and plan tasks.
  2. Pay attention to detail.
  3. Communicate better.
  4. Breakdown work structure.
I would also use IT tools like Slack and Trello more readily in order to help me organize, track and break down work structure.

3.4. To test digital solutions you must:

  1. Choose your target market / audience.
  2. What devices and software are used to access your site.
  3. Establish functions of the device / browser, for example, printing, page zooming, page sizing, copy/ paste, etc.
  4. Check for error messages, pop ups, pop out pages, overlays.
  5. Multiple session log ins.
